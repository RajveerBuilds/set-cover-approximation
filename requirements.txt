# Set Cover: Greedy Approximation vs Optimal (NP-hard)

This project studies the Set Cover problem, a classic NP-hard optimization problem, by comparing a fast greedy approximation algorithm with an exact brute-force solver on small instances.

## What this shows
- Why Set Cover is NP-hard and becomes computationally infeasible to solve optimally as input size grows.
- How greedy approximation provides fast, near-optimal solutions in practice.
- The trade-off between runtime and solution quality.

## Problem statement
Given a universe of elements and a collection of subsets, the goal is to select the smallest number of subsets whose union covers the entire universe.

## Approach
Two solvers are implemented:

- Brute-force (exact): Tries all possible combinations of subsets to find the optimal solution. This has exponential time complexity and is only feasible for very small instances.
- Greedy approximation: Repeatedly selects the subset that covers the maximum number of currently uncovered elements. This runs fast and scales to larger instances but does not always produce the optimal solution.

## Experiments
Random Set Cover instances are generated for increasing problem sizes.  
For small instances, both brute-force and greedy solutions are computed.  
For larger instances, only the greedy algorithm is evaluated due to the exponential cost of brute force.

The experiments visualize:
- Solution quality (number of sets selected)
- Runtime as problem size increases

## How to run
```bash
pip install -r requirements.txt
python main.py --max_n 15

This will generate two plots in the project directory:

- `solution_quality.png` — compares the number of sets selected by the greedy algorithm with the optimal (brute-force) solution for small instances.
- `runtime_comparison.png` — compares how runtime grows with problem size for greedy vs brute-force solvers.

## Observations
As the problem size increases, the exact solver becomes impractically slow due to exponential growth in computation time, while the greedy algorithm continues to run efficiently. The greedy solution may use more sets than the optimal solution, illustrating the trade-off between solution quality and computational efficiency.

## Why this project
This project demonstrates how approximation algorithms are used for NP-hard problems in practice and how empirical evaluation can reveal trade-offs between optimality and scalability, which are central themes in algorithm design and optimization.
